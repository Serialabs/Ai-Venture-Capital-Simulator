Methods

How this static simulation works

This project is static by design so it can run reliably on GitHub Pages with no backend dependencies. Inputs and outputs can still be curated manually today, while real-time and dynamic tooling can be added in later phases.

Fictional simulation: This workflow is a structured thought exercise, not an actual investment committee.

Not real investor quotes: Persona responses are synthetic and style-based, not verbatim statements.

Confidentiality reminder: Do not upload or publish sensitive, private, or regulated information.

4-stage pipeline

1. Parse the deal brief

Capture core facts, assumptions, unknowns, and explicit debate triggers from the startup input.

2. Run persona evaluation

Generate concise individual reads so each investor lens surfaces distinct upside and risk signals.

3. Conduct committee debate

Assign Bull, Bear, and Wild Card roles to pressure-test claims with escalating specificity.

4. Synthesize decision memo

Summarize key insights, decision posture, follow-up questions, and practical next actions.

Debate

Debate structure

Bull

Argues why the opportunity could compound and what conditions make it investable.

Bear

Challenges assumptions, stresses downside cases, and tests evidentiary gaps.

Wild Card

Introduces non-obvious vectors, edge scenarios, and second-order consequences.

  • Round 1: Frame first positions and define the central thesis conflict.
  • Round 2: Introduce concrete support, counterpoints, and evidence quality checks.
  • Round 3: Probe model durability across execution, market, and timing risks.
  • Round 4: Narrow disagreements to the highest-impact decision variables.
  • Round 5: State final conditional verdicts and what would change each view.
