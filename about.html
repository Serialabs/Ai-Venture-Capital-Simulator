How this static simulation works

This site is intentionally static so it can run on GitHub Pages. Reports are hand-curated and posted as structured articles, not generated in real time.

Fictional simulation: this is a framework to think through startup investment decisions.

Not real statements: persona outputs are interpretive writing, not real investor quotes.

Confidentiality: never post private company data on public pages.

4-stage analysis pipeline

  1. Structured digest: summarize core claims and supporting evidence by section/page.
  2. 10 persona evaluations: each persona returns verdict, strengths, concerns, crux, and change-my-mind conditions.
  3. 5-round debate: Bull / Bear / Wild Card debate with rebuttal requirements each round.
  4. Final synthesis: executive summary, key insights, founder questions, and action plan.

Debate structure

Bull

Argues upside and strategic optionality from the best available evidence.

Bear

Focuses on failure modes, missing proof, and downside asymmetry.

Wild Card

Introduces second-order effects and non-obvious decision constraints.

  • Round 1: framing and initial stances
  • Round 2: evidence challenge
  • Round 3: assumption stress test
  • Round 4: scenario sensitivity
  • Round 5: decision conditions and confidence
